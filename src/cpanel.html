cPanel Installation Guide

LaraPass v1 Installation Guide for cPanel powered Shared/VPS Hosting Servers.

  • Downloading
  • Setup cPanel
  • Uploading
  • Setup Database
  • Installing

Downloading

Download all the files you get after purchasing LaraPass v1 from CodeCanyon (Envato).

{primary} Keep the License File (with License/Purchase Code) safely as you will need it to activate LaraPass v1.

Extract all the files into a safe location. Also extract the laravel_source zip_file into a folder (it contains all the main files we will be uploading to our server).

Setup cPanel

Open cPanel in your browser, usually available at the url address yourdomain.com:2082 .

Navigate to your Domains, Addon Domains or Sub-domains depending on which type of domain you want to install LaraPass v1 on. For this example we will be using a sub-domain, so click on the Sub-Domains Page in your cPanel (the proceedure is almost same for all types of domains).

  • Pick a new subdomain, in our case, demo

  • Select the domain name, in our case, LaraPass v1.net

  • Document Root - LaraPass v1/public (remove the default document root that pops up)

  • Click Create

{info} Becareful - LaraPass v1 app should always be placed in or point towards a directory in the root/home directory of your server and not be accessible using the publicly available directory

Uploading

Open File Manager in cPanel or login to your server using an FTP program such as FileZilla or CyberDuck.

Navigate to the LaraPass v1 directory and open it (should be present in the root or home directory).

Upload all the files & folders you got from the laravel_source zip_file into the server directory.

{warning} Becareful not to miss any file or interrupt when uploading to server.

Setup Database

Open MySQL Databases in your cPanel.

  • Create a new database.

  • Create a new user (with a unique password).

  • Add the newly created user to the LaraPass v1 database and

  • add ALL PRIVILIGES to that user.

{success} Note down the Database Name, Username and Password to be used in the next step for Installation.
