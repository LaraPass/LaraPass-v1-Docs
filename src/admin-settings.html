Setting / Configuration Options

Configuration options available for Admins in LaraPass v1.

  • Application Settings
  • Mailer Settings
  • Artisan Commands
  • Access Mode
  • Maintenance Mode

Application Settings

List of various application settings you can change from the admin menu.

  • App Name [The name of your application]
  • App Description [The description of your application]
  • App Email (Main Email ex: support@yourdomain.com)
  • App Transactional Email (Email used for sending transactional email, ex: no-reply@yourdomain.com)
  • Default Membership Name [Default name for free membership type]
  • Google reCaptcha Site Key [Get the site key from Google.com/recaptcha]
  • Google reCaptcha Secret Key [Get the secret key from Google.com/recaptcha]
  • App Logo Coloured [Main coloured logo, use recommended size]
  • App Logo White [White Coloured logo used on the Login/Register pages, use recommended size]
  • App Favicon [Favicon for your application, recommended size 32x32px]
  • About Us Link [Link to your about us page]
  • Privacy Policy Link (default) [Using default shows the build-in privacy page or you can use a full url to link to an external privacy page]
  • Terms of Service Link (default) [Using default shows the build-in terms page or you can use a full url to link to an external terms page]

{warning} It is recommended not to keep changing the LaraPass v1 system settings often without proper understanding of the underlying system.

Mailer Settings

List of various mailer settings you can change from the admin menu.

  • Mailer Driver [The Driver you want to use for the Mailer, Ex: smtp, mailgun, etc]
  • Mailer Sender Address [The Email Address that will be showed to the Users as the senders address, Ex: support@yourdomain.com]
  • Mailer Sender Name [The Name that will be showed to the Users as the Mailers Name, Ex: YourApp]
    • # Mail Through SMTP
  • Mailer Hostname [Hostname of the Mailer, Ex: smtp.mailgun.com (Ignore in case of using mailgun API)]
  • Mailer Port [Port to be used by the mailer, Ex: 587 (Ignore in case of using mailgun API)]
  • Mailer Encryption [Encryption system for Mailer, Ex: tls (Only in case of smtp)]
  • Mailer Username [Your Mail Username, ex: smtp@yourdomain.com (Only in case of smtp mail driver)]
  • Mailer Password [Your Mail Password, ex: mypass2322ssawd (Only in case of smtp mail driver)]
    • # Mail Through Mailgun API
  • Mailgun Domain [Your Mailgun Domain Name, i.e., the domain you used to setup mailgun, Ex: mailgun.yourdomain.com]
  • Mailgun Secret [Your Mailgun Secret Key which you will get after successfully setting up mailgun, Ex: key-dawaw221dawdadw213132]

{warning} It is recommended not to keep changing the LaraPass v1 system settings often without proper understanding of the underlying system.

Laravel Artisan Commands

Laravel has a build in caching system that automatically & periodically caches various aspect of the application in order to serve users faster (caching improves response times), however whenever you make a change, you need to clear that old cache so the updated information is refelected throughout the app, as such we have build the Artisan Commands for clearing various caches right in the UI itself for ease. Available list of commands are -

PHP ARTISAN CACHE:CLEAR - This command button in the UI is used to clear system cached files.
PHP ARTISAN CONFIG:CLEAR - This command button in the UI is used to clear configuration files cached by the system.
PHP ARTISAN VIEW:CLEAR - This command button in the UI is used to clear view files cached by the system.
PHP ARTISAN ROUTE:CLEAR - This command button in the UI is used to clear routes file cached by the system.
PHP ARTISAN CLEAR-COMPILED - This command button in the UI is used to clear all compiled files from the system cache.
PHP ARTISAN VIEW:CACHE - This command button in the UI is used to cache fresh view files in the system.

{warning} It is recommended clear all cache files whenever you make any change to the system settings or to the code.

Access Mode

You can now switch your LaraPass v1 App between Public & Private Mode.

PUBLIC MODE - User Registrations Open - Anyone user with a valid email address can register and user your LaraPass v1 application in Public Mode.

PRIVATE MODE - User Registrations Closed - User Registrations are closed once the Application in set to Private Mode. Only the Admin can create new User Accounts from the Admin Overview Page.

{warning} Note: All User Accounts created before changing access mode to Private will still function normally. Only New Account Registrations will be restricted.

Maintenance Mode

Maintenance Mode now Available in LaraPass v1. You can click on the 'Activate Maintenance Mode' button in the Admin Settings Page and the Application will be put in Maintenace Phase, i.e., all users will be shown a custom maintenance page (with a custom message) for the duration of the maintenance.

Once the maintenance is complete, you can Deactivate it and the application will be set back to normal.

Admins can provide Custom Messages when Activating Maintenance Mode for better communication with their users.

Maintenance Mode is automatically activated and deactivated by the Updates Manager during an Update.

{info} Note: Your (the Admin User) IP Address will automatically be added to a whitelist when activating Maintenance Mode, so that you can still access the application without any hinderance.
